# LeetCode 76. 最小覆盖子串（Minimum Window Substring）

---

# 一、题目描述
![alt text](<../../Snap_for_Questions/4_Substring_子串/截屏2026-06-07 16.42.09.png>)

```python
class Solution:
    def minWindow(self, s: str, t: str) -> str:
        # 我们需要用Counter
        # 题目给我们 字符串：s，t， 长度m，n 空字符串""
        
        # 用来存储所需的检测的值和个数 的哈希表
        need = Counter(t)
        # 有出事默认值0 所以需要 defaultdic() 来构造初始哈希表
        window = defaultdict(int)

        # 滑动窗口默认初始法
        left = 0
        right = 0

        # 用来存储比较值的变量
        valid = 0
        # 用来记录从窗口最左往右 开始 收缩答案 找最佳的时候的 变量
        start = 0
        # 默认为 最大值，正无穷
        min_len = float('inf')

        # 开始 滑动窗口 标准模板
        while right < len(s):
            # 存储字符
            c = s[right]
            # 开始滑动
            right += 1

            # 判断条件
            if c in need:
                # 如果在need中，则对于我们自己的window进行该 键值对 的创建增加
                window[c] += 1
                # 判断此时的这个字符的 winodw 和 need 是否已经相匹配（键和值 都相等）
                if window[c] == need[c]:
                    valid += 1
            
            # 判断在 该字符加入后，是否总的 valid 和 len(need) 相等，是否找够了我们所需的子串
            while valid == len(need):
                
                # 开始判断并更新最优值-最短的子串，次数没有+-1因为，此时right已经移动到下一位了。
                if right - left < min_len:
                    start = left
                    min_len = right - left

                d = s[left]
                left += 1
                # 删除此时左边的这个值,要判断是否在所需的哈希表中
                if d in need:
                    # 如果在，但是我们要缩短窗口，那么必须同时减少 valid 和 删除 window[d]
                    if window[d] == need[d]:
                        valid -= 1
                    window[d] -= 1
        
        # 判断是否没有改变min_len 则输出空字符串, 否则 输出s 字符串的对应的位置
        return "" if min_len == float('inf') else s[start:start+min_len]


                    
```


给定两个字符串：

```python
s
t
```

要求在字符串 `s` 中找到一个最短连续子串，使得：

```text
该子串包含 t 中所有字符

并且字符出现次数也满足要求
```

如果不存在这样的子串：

```python
return ""
```


# 十、LeetCode 核心代码模式

这是面试最常见写法。

```python
from collections import Counter, defaultdict

class Solution:
    def minWindow(self, s: str, t: str) -> str:

        need = Counter(t)
        window = defaultdict(int)

        left = 0
        right = 0

        valid = 0

        start = 0
        min_len = float("inf")

        while right < len(s):

            c = s[right]
            right += 1

            if c in need:

                window[c] += 1

                if window[c] == need[c]:
                    valid += 1

            while valid == len(need):

                if right - left < min_len:
                    start = left
                    min_len = right - left

                d = s[left]
                left += 1

                if d in need:

                    if window[d] == need[d]:
                        valid -= 1

                    window[d] -= 1

        return "" if min_len == float("inf") else s[start:start+min_len]
```

```text
Step1 构造需要匹配的字符表
need = Counter(t)
比如：
need = {
    'A': 2,
    'B': 1,
    'C': 1
}
意思：
A 需要2个
B 需要1个
C 需要1个

Step2 构造窗口统计表
window = defaultdict(int)
用于统计当前窗口中的字符数量。

Step3 初始化变量
left = 0
right = 0

窗口：[left,right)
注意：这里采用：左闭右开写法

valid = 0
表示：已经满足要求的字符种类数


min_len = float("inf")
表示：
把 min_len 初始化为无穷大（∞）

Step4 右边扩张窗口
c = s[right]
right += 1
取出新字符：c 加入窗口。

Step5 更新窗口统计
if c in need:只有目标字符才处理。

window[c] += 1 统计增加。

Step6 判断是否满足要求
if window[c] == need[c]:
    valid += 1

Step7 判断窗口是否已经覆盖 t
核心：
while valid == len(need):

例如：
need={
'A':1,
'B':1,
'C':1
}
窗口：
window={
'A':2,
'B':1,
'C':1
}
此时：valid = 3
而：len(need)=3
说明：ABC全部满足
窗口已经合法。

开始收缩。

Step8 更新最优答案
if right-left < min_len:
    start = left
    min_len = right-left

记录：

最短合法窗口

Step9 左边收缩窗口
d = s[left]
left += 1
删除左边字符。

Step10 更新统计
if d in need:
如果是目标字符。

先判断：

if window[d] == need[d]:

例如：删除前：
window['A']=1
need['A']=1
说明：
A刚好满足
如果删掉：
A
那么：A不满足了
所以：valid -= 1
然后：window[d] -= 1
真正删除。


最后返回答案
return "" if min_len == float("inf") \
       else s[start:start+min_len]
如果：
min_len
从未更新。
说明：
不存在覆盖子串
返回：""
否则：s[start:start+min_len]
返回最短窗口。

面试记忆口诀
need    -> 目标频率表
window  -> 窗口频率表
valid   -> 已满足字符种类数

right扩张收集字符
valid满足开始收缩

更新答案
删除左边字符
维护valid

最终得到最短合法窗口

```
