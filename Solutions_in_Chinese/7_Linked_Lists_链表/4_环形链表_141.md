# LeetCode 141. 环形链表（Linked List Cycle）

---

# 一、题目描述
![alt text](<../../Snap_for_Questions/7_Linked_Lists_链表/截屏2026-06-24 18.06.04.png>)

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        # 我们需要双指针，需要其中的快慢双指针，因为快慢移动，如果有环，最终必会相遇。
        fast = head
        slow = head

        # 开始循环，固定的条件
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            
            # 如果引用地址发生了相等，如果移动必定有环 让他们成功相遇了。
            if slow == fast:
                return True
        return False
```
给你一个链表的头节点 `head`。

判断链表中是否存在环。

如果链表中有某个节点，可以通过不断执行：

```python
node = node.next
```

最终再次回到该节点。

则说明链表存在环。

返回：

```python
True
```

否则返回：

```python
False
```

输出：

```

```

```text

```

# 二、什么是环？

正常链表：

```text
1 → 2 → 3 → 4 → null
```

最终会到：

```python
None
```

---

环形链表：

```text
1 → 2 → 3 → 4
    ↑       ↓
    ← ← ← ← ←
```

永远不会到：

```python
None
```

会一直循环。

---

# 三、本题核心思想

最优解：

## Floyd 判圈算法（快慢指针）

又称：

```text
Floyd Cycle Detection
龟兔赛跑算法
```

核心思想：

```text
慢指针一次走一步

快指针一次走两步
```

如果有环：

```text
快指针一定会追上慢指针
```

如果没有环：

```text
快指针一定先到 None
```

---

# 四、为什么快指针一定能追上慢指针？

假设：

```text
环长度 = 5
```

进入环后：

```text
slow 每次 +1

fast 每次 +2
```

速度差：

```text
2 - 1 = 1
```

即：

```text
每轮都会缩短一个节点距离
```

最终：

```text
一定相遇
```

类似：

```text
跑道上两个人跑步

一个每秒跑1米

一个每秒跑2米

一定会追上
```

# 十二、另一种解法：HashSet

思路：

记录访问过的节点。

如果再次访问：

```text
说明有环
```

---

代码：

```python
class Solution:
    def hasCycle(self, head):

        visited = set()

        curr = head

        while curr:

            if curr in visited:
                return True

            visited.add(curr)

            curr = curr.next

        return False
```

---

## 为什么不推荐？

时间：

```text
O(n)
```

空间：

```text
O(n)
```

而快慢指针：

```text
O(1)
```

空间更优。

# 易错情况

```text
 if fast == slow:
        return True

    fast = fast.next.next
    slow = slow.next

return False
```
为什么这样的顺序会错？

原因一句话：因为初始化时，fast 和 slow 就指向同一个节点（head）。


正确版本注意：
先移动，再比较。

```text

为什么一定要"先走再比较"？

因为：初始化：

fast = head
slow = head

只是表示：两个人站在同一起跑线。

不能说明：两个人在比赛中相遇了。

真正要判断的是：移动之后有没有再次相遇。
```
fast 和 slow 初始化时本来就指向同一个 head，因此不能一开始就比较，否则所有非空链表都会被误判为有环。必须先让它们移动，再检查它们是否再次相遇，这样 fast == slow 才真正表示在环中追上了对方。
