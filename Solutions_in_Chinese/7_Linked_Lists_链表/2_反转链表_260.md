# LeetCode 206. 反转链表（Reverse Linked List）

## 一、题目描述
![alt text](<../../Snap_for_Questions/7_Linked_Lists_链表/截屏2026-06-22 17.43.38.png>)

直接上答案：
```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def reverseList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        # 使用双指针
        # 前一个节点，和当前节点 - 在脑海中构建链表图
        prev = None
        curr = head

        # 双指针开始按算法移动，循环条件：只要当前节点不为空，就继续反转
        while curr:
            # 1. 保存后面的链表地址引用
            next_node = curr.next
            # 2. 改变当前链表的指针方向 (反转)
            curr.next = prev
            # 3. 分别移动这两个指针
            prev = curr
            curr = next_node
        # prev 最终会指向 从末尾 - 开头。也就是满足要求反转了
        return prev
```

给你一个单链表的头节点 `head`。

请你反转链表，并返回反转后的链表。

例如：

原链表：

```
1 → 2 → 3 → 4 → 5 → null
```

反转后：

```
5 → 4 → 3 → 2 → 1 → null
```

---

# 二、核心思想

反转链表的本质：

> **改变每一个节点的 next 指针方向。**

原来的方向：

```
1 → 2 → 3 → 4 → 5 → null
```

反转之后：

```
null ← 1 ← 2 ← 3 ← 4 ← 5
```

最终：

```
5 → 4 → 3 → 2 → 1 → null
```

---

# 三、最经典方法：双指针迭代法（★★★★★）

这是 LeetCode 面试最常考的方法。

## 核心思想

使用两个指针：

```
prev：保存当前节点的前一个节点
curr：正在处理的当前节点
```

初始状态：

```
prev        curr
 ↓           ↓
null        1 → 2 → 3 → 4 → 5
```

每一次循环完成三件事：

1. 保存当前节点的下一个节点
2. 反转当前节点的指针方向
3. 两个指针向后移动

---

# 四、反转过程图解

## 第一次循环

初始：

```
prev        curr
 ↓           ↓
null         1 → 2 → 3 → 4 → 5
```

### ① 保存后面的节点

```python
next_node = curr.next
```

结果：

```
next_node
    ↓
    2 → 3 → 4 → 5
```

为什么需要保存？

因为马上要修改：

```python
curr.next
```

如果不提前保存，后面的链表会丢失。

---

### ② 反转指针

```python
curr.next = prev
```

结果：

```
null ← 1      2 → 3 → 4 → 5
       ↑
      curr
```

---

### ③ 移动两个指针

```python
prev = curr
curr = next_node
```

结果：

```
prev       curr
 ↓          ↓
1 → null    2 → 3 → 4 → 5
```

---

## 第二次循环

保存：

```
next_node = 3
```

反转：

```
2 → 1 → null
```

移动：

```
prev       curr
 ↓          ↓
 2 → 1      3 → 4 → 5
```

---

不断重复，直到：

```
curr = null
```

最终：

```
prev
 ↓
 5 → 4 → 3 → 2 → 1 → null
```

---

# 五、LeetCode 核心代码模式（必须背）

## 代码模板

```python
class Solution:
    def reverseList(self, head: ListNode) -> ListNode:

        # 前一个节点
        prev = None

        # 当前节点
        curr = head


        # 当前节点不为空，就继续反转
        while curr:

            # 1. 保存后面的链表
            next_node = curr.next

            # 2. 改变指针方向
            curr.next = prev

            # 3. 移动两个指针
            prev = curr
            curr = next_node


        # prev 最终指向新的头节点
        return prev
```

# 六、逐句代码讲解

---

## 1. 初始化前驱节点

```python
prev = None
```

作用：

表示当前节点前面没有节点。

因为反转之后：

```
1 → null
```

第一个节点会变成尾节点，它的 next 必须指向 `None`。

---

## 2. 初始化当前节点

```python
curr = head
```

作用：

从原链表的头节点开始遍历。

例如：

```
curr
 ↓
 1 → 2 → 3 → 4
```

---

## 3. 循环遍历链表

```python
while curr:
```

等价于：

```python
while curr != None:
```

含义：

只要当前节点存在，就继续进行反转。

---

## 4. 保存后续节点

```python
next_node = curr.next
```

作用：

保存当前节点后面的链表。

为什么必须保存？

因为下一步：

```python
curr.next = prev
```

会修改当前节点的 next。

如果没有保存：

```
1 → 2 → 3
```

修改：

```
1 → null
```

那么：

```
2 → 3
```

这部分就无法访问了。

---

## 5. 改变指针方向

```python
curr.next = prev
```

这是整个算法最核心的一步。

原来：

```
prev   curr
 ↓      ↓
null ←  1 → 2
```

执行后：

```
null ← 1
       ↑
      curr
```

让链表方向反过来。

---

## 6. 移动 prev

```python
prev = curr
```

作用：

让 prev 永远指向已经反转完成的链表头。

例如：

反转：

```
2 → 1 → null
```

此时：

```
prev
 ↓
 2 → 1
```

---

## 7. 移动 curr

```python
curr = next_node
```

作用：

继续处理剩下未反转的链表。

例如：

```
curr
 ↓
 3 → 4 → 5
```

---

## 8. 返回新的头节点

```python
return prev
```

为什么不是：

```python
return curr
```

因为：

循环结束时：

```
curr = None
```

例如：

```
5 → 4 → 3 → 2 → 1 → null
                ↑
              已经完成
```

此时：

```
prev
 ↓
 5 → 4 → 3 → 2 → 1
```

所以返回：

```python
prev
```

prev = None

# 八、递归方法（了解）

除了迭代，还可以使用递归。

## 思想

假设：

```
1 → 2 → 3 → 4 → 5
```

先反转：

```
2 → 3 → 4 → 5
```

得到：

```
5 → 4 → 3 → 2
```

然后把：

```
1
```

接到最后。

---

## LeetCode 递归代码

```python
class Solution:
    def reverseList(self, head: ListNode) -> ListNode:

        # 递归终止条件
        if not head or not head.next:
            return head


        # 反转后面的链表
        new_head = self.reverseList(head.next)


        # 改变方向
        head.next.next = head


        # 原来的头节点变成尾节点
        head.next = None


        return new_head
```

---

# 九、递归代码逐句解析

## 1. 递归终止条件

```python
if not head or not head.next:
    return head
```

两种情况：

### 空链表：

```
None
```

### 只有一个节点：

```
1 → null
```

它本身就是反转后的结果。

---

## 2. 递归反转后面的链表

```python
new_head = self.reverseList(head.next)
```

例如：

当前：

```
1 → 2 → 3
```

递归：

```
2 → 3
```

返回：

```
3 → 2
```

---

## 3. 让后面的节点指向自己

```python
head.next.next = head
```

例如：

原来：

```
1 → 2
```

递归后：

```
2 → null
```

执行：

```
2 → 1
```

---

## 4. 切断原来的连接

```python
head.next = None
```

为什么？

避免形成环：

错误：

```
1 ↔ 2
```

正确：

```
2 → 1 → null
```

---

## 5. 返回新的头节点

```python
return new_head
```

递归过程中：

新的头节点一直是原链表的尾节点。

# 十、ACM 模式写法（面试常用）

def reverseList(head):

# 十二、复杂度分析

## 方法一：迭代双指针

### 时间复杂度

```
O(n)
```

原因：

每个节点访问一次。

---

### 空间复杂度

```
O(1)
```

只使用：

```
prev
curr
next_node
```

三个额外指针。

---

## 方法二：递归

### 时间复杂度

```
O(n)
```

---

### 空间复杂度

```
O(n)
```

因为递归调用栈需要保存每一层函数状态。

# 十三、链表反转核心模板总结（★★★★★）

## 1. 迭代模板

```python
prev = None
curr = head

while curr:
    next_node = curr.next
    curr.next = prev
    prev = curr
    curr = next_node

return prev
```

适用于：

- 206. 反转链表
- 92. 反转链表 II
- K 个一组翻转链表
- 链表排序中的局部翻转

---

## 2. 递归模板

```python
def reverse(head):

    if not head or not head.next:
        return head

    new_head = reverse(head.next)

    head.next.next = head

    head.next = None

    return new_head
```

# 最终总结

这道题是 LeetCode 链表专题中最重要的基础题之一。

其核心思想只有一句：

```
让当前节点的 next 指向前一个节点，
并不断向后移动，直到链表结束。
```

必须熟练掌握的核心代码：

```python
while curr:
    next_node = curr.next
    curr.next = prev
    prev = curr
    curr = next_node
```

它是后续所有高级链表题（局部反转、K 个一组翻转、回文链表等）的基础模板。
