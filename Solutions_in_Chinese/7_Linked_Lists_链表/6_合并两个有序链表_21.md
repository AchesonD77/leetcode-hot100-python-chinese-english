# LeetCode 21. 合并两个有序链表（Merge Two Sorted Lists）

---

# 一、题目描述

![alt text](<../../Snap_for_Questions/7_Linked_Lists_链表/截屏2026-06-26 19.13.43.png>)

快速答案：
```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def mergeTwoLists(self, list1: Optional[ListNode], list2: Optional[ListNode]) -> Optional[ListNode]:
        # 我们使用归并递归思想，需要一个dummy虚拟节点头和当前尾节点的双指针：
        dummy = ListNode()
        cur = dummy

        # 开始循环
        while list1 and list2:
            # 进行比较，找最小接到链表上
            if list1.val <= list2.val:
                cur.next = list1
                list1 = list1.next
            else:
                cur.next = list2
                list2 = list2.next
            
            cur = cur.next
        # 街上当其中某个为空的情况下，只剩另一个有序链表的情况
        cur.next = list1 if list1 else list2

        return dummy.next

```

给你两个**按升序排列**的单链表：

```text
list1
list2
```

要求：

将它们合并成一个新的**升序链表**，并返回新的头节点。

注意：

- **不能新建节点保存数据**
- **只能修改节点之间的 next 指针**
- 返回新的链表头节点

例如：

```
list1:

1 → 2 → 4

list2:

1 → 3 → 4
```

合并后：

```
1 → 1 → 2 → 3 → 4 → 4
```

# 六、LeetCode 核心代码模式（★★★★★）

```python
class Solution:
    def mergeTwoLists(
        self,
        list1: ListNode,
        list2: ListNode
    ) -> ListNode:

        # 虚拟头节点
        dummy = ListNode()

        # 当前尾节点
        cur = dummy


        # 两个链表都还有节点
        while list1 and list2:

            if list1.val <= list2.val:

                cur.next = list1
                list1 = list1.next

            else:

                cur.next = list2
                list2 = list2.next

            cur = cur.next


        # 接上剩余链表
        cur.next = list1 if list1 else list2

        return dummy.next
```

很多人第一次学链表都会问：

不是刚刚已经 cur.next = list1（或 list2）了吗？为什么还要 cur = cur.next？

答案一句话就是：

cur.next = ... 是把新节点接到链表后面，而 cur = cur.next 是把尾指针移动到这个新节点上，为下一次拼接做准备。

牢记一个原则：

cur.next = ... 是"接节点"，cur = cur.next 是"移动尾指针"。只有这两步配合，才能不断把新节点正确地接到结果链表的最后面。

其实这道题也是经典递归题。

## LeetCode 递归模板

```python
class Solution:
    def mergeTwoLists(self, list1, list2):

        if not list1:
            return list2

        if not list2:
            return list1

        if list1.val <= list2.val:

            list1.next = self.mergeTwoLists(
                list1.next,
                list2
            )

            return list1

        else:

            list2.next = self.mergeTwoLists(
                list1,
                list2.next
            )

            return list2
```

# 最终总结

本题的本质：

```
每次从两个有序链表的头节点中，

选择较小的一个接到新链表尾部，

直到其中一个链表为空，

最后将剩余部分整体接到新链表末尾。
```

必须熟练掌握的核心代码：

```python
dummy = ListNode()
cur = dummy

while list1 and list2:

    if list1.val <= list2.val:
        cur.next = list1
        list1 = list1.next
    else:
        cur.next = list2
        list2 = list2.next

    cur = cur.next

cur.next = list1 if list1 else list2

return dummy.next
```

这是 **LeetCode Hot100 链表专题中 Dummy（虚拟头节点）模板的经典入门题**，也是归并排序和后续高级链表题的重要基础，必须熟练掌握。

# 个人易错总结：

没有真正理解 cur 和 cur.next 到底代表什么。

- cur 永远表示"结果链表的尾节点（Tail）"，而 cur.next 表示"尾节点后面的那个位置（可以接新节点的位置）"。

| 指针         | 作用             | 如何移动                                    |
| ---------- | -------------- | --------------------------------------- |
| `list1`    | 遍历第一个链表        | `list1 = list1.next`                    |
| `list2`    | 遍历第二个链表        | `list2 = list2.next`                    |
| `cur`      | 指向结果链表尾部       | `cur = cur.next`                        |
| `cur.next` | 尾部后面的空位置，用来接节点 | `cur.next = list1` 或 `cur.next = list2` |

- list1、list2 是"读指针"（负责读取原链表）；cur 是"写指针"（负责维护结果链表的尾部）；cur.next 才是真正连接新节点的位置。因此永远是先 cur.next = 节点，再 cur = cur.next，绝不能写成 cur = list1.next。
