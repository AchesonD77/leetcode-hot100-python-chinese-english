直接答案：
```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def getIntersectionNode(self, headA: ListNode, headB: ListNode) -> Optional[ListNode]:
        # 我们创建两个遍历指针，（headA，B是单链表的头结点）设置初始的链表指向
        pA = headA
        pB = headB

        # 开始循环 找指向相等的节点地址，不相等继续循环 
        while pA != pB:
            
            pA = pA.next if pA else headB
            pB = pB.next if pB else headA

            ''' 旧的写法
            # 等价于 pA != null:
            if pA: 
                # 移动到链表下一位
                pA = pA.next
            # 当pA == null后，移动到B的链表头开始移动
            else:
                pA = headB

            if pB:
                pB = pB.next
            else:
                pB = headA
                '''
        return pA

```

给你两个单链表的头节点 `headA` 和 `headB`。

请你找出两个链表开始相交的第一个节点。

如果两个链表不存在相交节点，返回 `null`。

## 二、什么叫链表相交？

注意：

**链表相交比较的是节点地址，而不是节点值。**

例如：

```
A:
    4 → 1
        \
        8 → 4 → 5
        /
B:
5 → 6 → 1
```

这里节点值为 `8` 的节点开始相交。

也就是说：

```
A 的节点8 和 B 的节点8
指向的是同一个内存节点
```

而不是两个值都为 `8` 的不同节点。

---

## 三、核心思想：双指针走两遍链表（最优解）

### 1. 为什么长度不同会导致无法同时到达？

例如：

```
A: a1 → a2 → c1 → c2 → c3
长度 = 5

B: b1 → b2 → b3 → c1 → c2 → c3
长度 = 6
```

两个指针：

```
pA 从 A 出发
pB 从 B 出发
```

第一次同时移动：

```
pA 走：
a1 a2 c1 c2 c3 null

pB 走：
b1 b2 b3 c1 c2 c3 null
```

由于两个链表前面的长度不同，它们无法同时到达交点。

---

### 2. 交换链表继续走

当：

```
pA 到 null
```

让它去：

```
headB
```

当：

```
pB 到 null
```

让它去：

```
headA
```

这样：

```
pA 走过的路径：
A + B

pB 走过的路径：
B + A
```

两者总路程完全一样：

```
长度A + 长度B
```

所以：

- 有交点：一定会在交点相遇
- 没交点：最终会同时到达 null

---

## 四、双指针移动过程示意

假设：

```
A:
a1 → a2 → c1 → c2 → c3

B:
b1 → b2 → b3 → c1 → c2 → c3
```

第一轮：

```
pA:
a1 → a2 → c1 → c2 → c3 → null

pB:
b1 → b2 → b3 → c1 → c2 → c3 → null
```

交换：

```
pA:
b1 → b2 → b3 → c1

pB:
a1 → a2 → c1
```

最终：

```
pA == pB == c1
```

找到交点。

# 五、LeetCode 核心代码模式

## 代码模板

```python
class Solution:
    def getIntersectionNode(
        self,
        headA: ListNode,
        headB: ListNode
    ) -> ListNode:

        pA = headA
        pB = headB

        while pA != pB:

            if pA:
                pA = pA.next
            else:
                pA = headB


            if pB:
                pB = pB.next
            else:
                pB = headA

        return pA
```

---

## 六、逐句代码讲解

### 1. 初始化两个指针

```python
pA = headA
pB = headB
```

作用：

创建两个遍历指针。

初始位置：

```
pA:
A 链表头

pB:
B 链表头
```

---

### 2. 循环条件

```python
while pA != pB:
```

含义：

只要两个指针没有指向同一个节点，就继续移动。

注意：

这里比较的是：

```
节点地址
```

不是：

```
节点值
```

例如：

```python
pA.val == pB.val
```

不能说明它们相交。

必须：

```python
pA == pB
```

才表示两个指针指向同一个节点。

---

### 3. 移动 pA

```python
if pA:
    pA = pA.next
else:
    pA = headB
```

分情况：

---

#### 情况1：没有走到末尾

例如：

```
pA
 ↓
a1 → a2 → c1
```

执行：

```python
pA = pA.next
```

移动到下一个节点。

---

#### 情况2：已经走到末尾

例如：

```
a1 → a2 → c1 → null
                 ↑
                 pA
```

执行：

```python
pA = headB
```

重新从 B 链表开始。

---

### 4. 移动 pB

代码完全相同：

```python
if pB:
    pB = pB.next
else:
    pB = headA
```

作用：

让 pB：

```
B → A
```

完成另一条路径。

---

### 5. 返回结果

链表最核心的概念：节点（对象）、值（val）、地址（引用）之间的区别。

先给结论：
在 Python 的 LeetCode 链表中，return pA 返回的是 节点对象的引用（可以理解为地址），不是单纯返回节点里面的值 val。

```python
return pA
```

循环结束只有两种情况：

---

情况一：

```
pA == pB == 交点
```

返回相交节点。

---

情况二：

```
pA == pB == None
```

说明没有交点。

返回：

```python
None
```

# 八、ACM 模式写法（面试常用）

在 ACM 模式中，通常输入链表后，需要实现一个函数。

面试/ACM 常用：数组转换成链表

因为输入通常是：
nums = [1,2,3,4]
我们需要一个函数转换：
```python
def createLinkedList(nums):
    dummy = ListNode(0)
    cur = dummy

    for x in nums:
        cur.next = ListNode(x)
        cur = cur.next

    return dummy.next
```

LeetCode 后台已经帮你完成：
```
headA = createLinkedList(...)
headB = createLinkedList(...)
```
如何打印链表：
写一个打印函数：
```python
def printLinkedList(head):
    cur = head

    while cur:
        print(cur.val, end=" -> ") # 访问当前节点值
        cur = cur.next # 访问下一个节点

    print("None")
```
使用方法：

head = createLinkedList([1,2,3,4])

printLinkedList(head)

1 -> 2 -> 3 -> 4 -> None

 [4] → [1]

pA

 [4]

## 2. 判断对象是否为空

Python 中：

```python
if pA:
```

等价于：

```python
if pA != None:
```

表示当前节点存在。

---

## 3. 三元表达式

写法：

```python
a if 条件 else b
```

例如：

```python
pA = pA.next if pA else headB
```

等价于：

```python
if pA:
    pA = pA.next
else:
    pA = headB
```

# 总结

这道题的本质：

```
利用两个指针交换起点，
让它们走过相同长度的路径，
从而消除两个链表长度差。
```

核心代码只需要：

```python
while pA != pB:
    pA = pA.next if pA else headB
    pB = pB.next if pB else headA
```

这是 LeetCode Hot100 链表专题中最重要的双指针思想之一，必须熟练掌握。

为什么走到 A 末尾，要跳到 B？

```text
核心思想一句话

让两个指针走相同的总距离。

两个链表长度可能不同：
A: a + c
B: b + c
其中：
a = A 独有部分长度
b = B 独有部分长度
c = 相交部分长度

1. 初始化两个指针
代码：

pB = headB
表示：pA 从 A 头开始 pB 从 B 头开始
2. 同时往后走
代码：
if pA:
    pA = pA.next

意思：如果还没到尾部 继续向后走一个节点

重点来了：走到末尾怎么办？
代码：

else:
    pA = headB

也就是：
A 走完了
↓
跳到 B 的开头

同理：
else:
    pB = headA

表示：
B 走完了
↓
跳到 A 的开头

为什么这样就一定相遇？

这是最重要的地方。
pA 走的路线：
A + B
也就是：
a + c + b

pB 走的路线：
B + A
也就是：
b + c + a

两者总距离：
a + b + c
完全一样！

所以：
较长链表多出来的部分
会被第二轮抵消

双指针：
A走完走B
B走完走A

两人总路程：
A + B
B + A

距离相同，
长短差被抵消。

相遇：
返回交点

不相交：
同时到 None

```

    ↓

pA.next
    ↓
    获取下一个节点的引用
