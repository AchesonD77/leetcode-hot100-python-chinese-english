# 链表（Linked List）与头节点 `head` 详解

## 一、什么是链表（Linked List）

数组中的元素是连续存储的，可以通过下标快速访问。

例如：

```text
数组：

index:
  0    1    2    3

[10] [20] [30] [40]
```

访问：

```python
nums[2]
```

得到：

```text
30
```

---

但是链表不同。

链表中的元素并不是连续存储的，而是由一个个 **节点（Node）** 通过指针连接起来：

```text
10 → 20 → 30 → 40 → None
```

其中：

- `10`：第一个节点存储的值
- `20`：第二个节点存储的值
- `→`：表示指向下一个节点
- `None`：表示链表结束，没有下一个节点


---

# 二、什么是节点（ListNode）

链表中的每一个元素叫做一个节点。

每个节点包含两个部分：

```text
+----------------+
|      Node      |
+----------------+
| val  |  next   |
+----------------+
```

含义：

- `val`：当前节点保存的数据
- `next`：保存下一个节点的地址（引用）


Python 中通常定义：

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val      # 当前节点的值
        self.next = next    # 指向下一个节点
```

例如创建一个节点：

```python
node = ListNode(10)
```

内存理解：

```text
node
 |
 ↓
+-------------+
| val = 10    |
| next = None |
+-------------+
```

---

# 三、什么是头节点 `head`（最重要）

假设有一个链表：

```text
10 → 20 → 30 → 40 → None
```

真正的关系是：

```text
head
 ↓
[10 | next] → [20 | next] → [30 | next] → [40 | None]
```

这里：

```text
head
```

**不是数字 10。**

它保存的是：

```text
第一个节点的引用（可以理解为第一个节点的地址）
```

也就是：

```text
head
 ↓
第一个节点
```

---

# 四、为什么必须有 `head`

数组可以通过下标找到第一个元素：

```python
nums[0]
```

但是链表没有下标：

```text
10 → 20 → 30 → 40
```

每一个节点只知道：

```text
自己是谁
       ↓
下一个节点在哪里
```

例如：

```text
节点10：

val = 10
next → 节点20
```

它不知道：

- 自己是第几个节点
- 整个链表有多长
- 前面的节点在哪里


因此必须有一个入口：

```text
head
 ↓
10 → 20 → 30 → 40
```

通过 `head` 才能找到整个链表。


---

# 五、如何遍历链表

通常不会移动 `head`，而是创建一个辅助指针：

```python
cur = head
```

此时：

```text
head
 ↓
10 → 20 → 30 → 40
 ↑
cur
```

---

## 第一次循环

访问：

```python
cur.val
```

得到：

```text
10
```

然后：

```python
cur = cur.next
```

移动：

```text
head
 ↓
10 → 20 → 30 → 40
      ↑
     cur
```

---

## 第二次循环

访问：

```python
cur.val
```

得到：

```text
20
```

继续：

```python
cur = cur.next
```

变成：

```text
head
 ↓
10 → 20 → 30 → 40
           ↑
          cur
```

---

不断重复：

```python
cur = cur.next
```

最终：

```text
10 → 20 → 30 → 40
                ↑
               cur
```

再次移动：

```python
cur = cur.next
```

得到：

```text
cur = None
```

说明已经到达链表末尾。


---

## 链表遍历模板

```python
cur = head

while cur:
    print(cur.val)
    cur = cur.next
```

含义：

1. 从头节点开始
2. 访问当前节点的值
3. 移动到下一个节点
4. 直到 `cur == None`


---

# 六、为什么遍历时不要移动 `head`

原始：

```text
head
 ↓
10 → 20 → 30 → 40
```

如果写：

```python
head = head.next
```

变成：

```text
       head
        ↓
10 → 20 → 30 → 40
```

此时：

- 原来的节点 `10` 没有变量指向
- 失去了整个链表的入口


因此实际开发中：

```python
cur = head
```

然后移动：

```python
cur = cur.next
```

而：

```text
head 永远保存链表入口
```

---

# 七、链表常见变量总结

| 名称 | 作用 | 通俗理解 |
|---|---|---|
| `head` | 指向第一个节点 | 链表入口 |
| `cur` | 遍历链表的指针 | 一个移动的游标 |
| `val` | 当前节点保存的数据 | 节点里面的值 |
| `next` | 指向下一个节点 | 通往下一个节点的路 |
| `None` | 没有下一个节点 | 链表结束 |

---

# 八、创建一个完整链表

代码：

```python
a = ListNode(1)
b = ListNode(2)
c = ListNode(3)

a.next = b
b.next = c

head = a
```

形成：

```text
head
 ↓
1 → 2 → 3 → None
```

---

访问第一个节点：

```python
head.val
```

结果：

```text
1
```

---

访问第二个节点：

```python
head.next.val
```

过程：

```text
head
 ↓
1 → 2 → 3
     ↑
```

结果：

```text
2
```

---

访问第三个节点：

```python
head.next.next.val
```

结果：

```text
3
```

---

# 九、数组与链表对比（面试高频）

| 对比 | 数组 Array | 链表 Linked List |
|---|---|---|
| 存储方式 | 连续内存 | 分散内存，通过 `next` 连接 |
| 查找方式 | 下标访问 | 从 `head` 依次遍历 |
| 随机访问 | O(1) | O(n) |
| 插入删除 | 可能移动大量元素 | 修改指针即可 |

---

# 十、面试必须记住的核心思想

```text
数组靠下标找到元素。

链表没有下标，只能从 head 出发，
通过 next 一个一个向后寻找。
```

---

```text
Node = val + next

head = 链表入口（第一个节点的引用）

cur = 用来移动的指针

cur.val  = 获取当前节点的值

cur.next = 移动到下一个节点
```

---

# 最核心的一张图（牢记）

```text
                 head
                  |
                  ↓

+----------------+      +----------------+      +----------------+
| val = 1        | ---> | val = 2        | ---> | val = 3        |
| next = 地址A   |      | next = 地址B   |      | next = None    |
+----------------+      +----------------+      +----------------+

        ↑                       ↑                       ↑
      节点1                    节点2                  节点3
```

---

# 一句话总结（面试版）

> 链表本质是由一个个节点通过 `next` 指针连接形成的线性结构，`head` 是访问整个链表的入口，它保存第一个节点的引用。遍历链表时使用 `cur` 从 `head` 出发，不断执行 `cur = cur.next`，直到 `cur == None` 表示到达链表尾部。
