# LeetCode 142. 环形链表 II（Linked List Cycle II）

---

# 一、题目描述
![alt text](<../../Snap_for_Questions/7_Linked_Lists_链表/截屏2026-06-25 14.46.48.png>)
![alt text](<../../Snap_for_Questions/7_Linked_Lists_链表/截屏2026-06-25 14.47.01.png>)

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
        # 环形链表题，我们使用快慢双指针和弗洛伊德算法来进行盘圈，首先使用一次来找到相遇点
        # 第二次使用经典算法原理 = 相遇点和head头结点开始同时慢移动，终会在入口相遇。
        fast = head
        slow = head

        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            
            if fast == slow:
                floyd = head

                #  要再次使用双指针循环
                while floyd != slow:
                    floyd = floyd.next
                    slow = slow.next
                return floyd
        return None
```
给定一个链表的头节点 `head`。

如果链表中存在环，则返回**环的入口节点**；如果不存在环，则返回 `None`。

> **注意：**
>
> - 不允许修改链表。
> - 要求尽量使用 **O(1)** 额外空间。

例如：

```
3 → 2 → 0 → -4
    ↑       ↓
    └───────┘
```

返回：

```
节点 2
```

因为：

```
2
```

就是进入环的第一个节点。

# 三、核心思想（★★★★★）

这道题仍然使用：

> **Floyd 快慢指针算法（龟兔赛跑）**

整个算法分为两个阶段：

```
第一阶段：
快慢指针找到相遇点

↓

第二阶段：
重新从头开始寻找入口
```

很多同学觉得第二步非常神奇，其实背后的数学证明非常漂亮。

# 四、第一阶段：寻找相遇点

初始化：

```python
slow = head
fast = head
```

移动规则：

```
slow 一次走一步

fast 一次走两步
```

如果：

```
fast 到 None
```

说明：

```
没有环
```

否则：

```
一定会相遇
```

例如：

```
3 → 2 → 0 → -4
    ↑       ↓
    └───────┘
```

第一次：

```
slow -> 2

fast -> 0
```

第二次：

```
slow -> 0

fast -> 2
```

第三次：

```
slow -> -4

fast -> -4
```

相遇。

注意：

> **相遇点不是入口！**

很多人误以为：

```
第一次相遇的位置
=
入口
```

这是错误的。

# 五、为什么重新从头走就能找到入口？

这是整道题最重要的地方。

设：

```
a
=
头节点到入口距离

b
=
入口到相遇点距离

c
=
相遇点回入口距离
```

如下图：

```
Head
 |
 |<----- a ----->

入口
 ●──────────────●
 ↑              │
 │              │
 │              │
 ●──────────────●
 相遇点
```

环长度：

```
L = b + c
```

---

## 慢指针

走过：

```
a + b
```

---

## 快指针

走过：

```
a + b + nL
```

因为：

```
fast = 2 × slow
```

得到：

```
2(a+b)
=
a+b+nL
```

整理：

```
a+b=nL
```

即：

```
a=nL-b
```

又因为：

```
L=b+c
```

得到：

```
a=n(b+c)-b
```

继续整理：

```
a=(n-1)L+c
```

这说明：

```
从 Head 到入口

=

从相遇点继续走 c
```

因此：

> **一个指针从 Head 开始，一个指针从相遇点开始，两者同时每次走一步，一定会在入口相遇。**

这就是 Floyd 算法最经典的数学结论。

# 七、LeetCode 核心代码模式（★★★★★）

```python
class Solution:
    def detectCycle(self, head: ListNode) -> ListNode:

        slow = head
        fast = head

        # 第一阶段：寻找相遇点
        while fast and fast.next:

            slow = slow.next
            fast = fast.next.next

            if slow == fast:

                # 第二阶段：寻找入口
                ptr = head

                while ptr != slow:

                    ptr = ptr.next
                    slow = slow.next

                return ptr

        return None
```

# 十三、HashSet 解法（了解）

思路：

记录所有访问过的节点。

再次访问：

```
说明进入环
```

第一次重复访问的节点：

```
就是入口
```

代码：

```python
class Solution:
    def detectCycle(self, head):

        visited = set()

        curr = head

        while curr:

            if curr in visited:
                return curr

            visited.add(curr)

            curr = curr.next

        return None
```

---

## 为什么不推荐？

```

```

```
O(1)
```

空间更优。

# 最终总结

本题是 **Floyd 判圈算法** 的经典应用，也是链表专题中最重要的数学题之一。

算法分为两个阶段：

```
第一阶段：
快慢指针判断是否有环，并找到相遇点。

↓

第二阶段：
一个指针从头节点开始，
另一个指针从相遇点开始，
每次同时走一步，
再次相遇的位置就是环的入口。
```

必须熟练掌握的核心代码：

```python
slow = fast = head

while fast and fast.next:
    slow = slow.next
    fast = fast.next.next

    if slow == fast:

        ptr = head

        while ptr != slow:
            ptr = ptr.next
            slow = slow.next

        return ptr

return None
```

这是 LeetCode Hot100 链表专题中最经典、最具代表性的快慢指针算法之一，也是面试中出现频率极高的必考题。

# 易错总结
```
return None

                 detectCycle()

                       │
                       ▼
                快慢指针寻找相遇点
                       │
          ┌────────────┴────────────┐
          │                         │
     找到相遇点                 没找到相遇点
          │                         │
          ▼                         ▼
    Floyd算法找入口           while 循环结束
          │                         │
          ▼                         ▼
   return 入口节点             return None
```
很多人会问：

为什么 return None 放在 while 外面？为什么不放到 while 里面？

实际上，这一句表示的是：

"整个链表遍历结束了，都没有发现环，因此返回 None。"

return None 放在 while 外面，是因为它表示整个链表都已经检查完成。如果在循环过程中找到了快慢指针的相遇点，就会提前 return floyd；只有当 while 正常结束（即 fast 或 fast.next 为 None，说明链表没有环）时，程序才会执行 return None，表示不存在环，也就不存在环的入口节点。
